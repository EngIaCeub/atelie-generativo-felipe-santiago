# Processo Git/GitHub

1. Mapear marcos acadêmicos em issues e, se útil, milestones.
2. Definir responsável e critério do barema em cada issue.
3. Usar branches curtas e commits pequenos ao longo das semanas.
4. PR deve incluir resumo, arquivos, validações, evidências e riscos.
5. Não incluir token, dado pessoal, checkpoint pesado ou resultado fabricado.
6. Manter contribuições rastreáveis de todos os membros; se projeto individual, manter issues/commits
   que mostrem processo incremental.
7. Atualizar estado após merge e fechar issue apenas com critério de aceite comprovado.
